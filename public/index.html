API Documentation

Toys Routes ⬇️

  • Get a list of all the toys, will present 10 results per page /toys (GET)
  • Get a list of all the toys, will present 10 results in page 2 /toys/?-page=2 (GET)
  • Get X product per one page: (default: 9) /toys/?perPage=3 (GET)
  • Search porduct by name or information /toys/search/?s=fun (GET)
  • Get a list of all the toys in estrategy category, will present 10 results per page: /toys/category/estrategy (GET)
  • Get a list of toys of the range of price/toys/price/?max=100&min=5 (GET)

Add a new toy to the data base, You need a token for this query /toys (POST)

  • Name (required)
  • Info (required)
  • Category (required)
  • Color (required)
  • Price (required)
  • Img_url

Update a toy in the data base, You need a token for this querytoys/id (PUT)

  • Name (required)
  • Info (required)
  • Category (required)
  • Color (required)
  • Price (required)
  • Img_url

Delete a toy from the data base, You need a token for this query toys/id (DELETE)

users Routes ⬇️

Add a new user to the data base Sing Up: /users (POST)
  • Name (required)
  • Email (required)
  • Password (required)

User login, if the email and password are correct the server will respond with a token Sing In: /users/login (POST)
  • Email (required)
  • Password (required)
